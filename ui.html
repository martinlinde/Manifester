HUD Design System Sync

Import Design Tokens

Load your design-tokens.json file to create visual swatches and guides in Figma.

This will create a "HUD Design System" frame with:
  • Color swatches (editable fills)
  • Spacing scale visualization
  • Border radius samples
  • Typography size samples
  • Component height guides

Export Design Tokens

Extract colors, spacing, and other values from the "Component Demo" frame back to JSON.

After exporting:
  1. Save design-tokens.json to scripts/tools/exports/
  2. Refresh app - tokens load automatically

Generate Component Demo

Generate the master Component Demo panel in Figma. Edit colors and styles, then export tokens back to code.

1. Design Tokens (styling values)

2. Node Manifest (optional - structure from export)

Two Generation Modes:
  • Component Demo: Creates a fresh panel from semantic structure (sections + controls)
  • Regenerate Exact: Recreates the EXACT exported structure with all positions, sizes, and layouts (requires node-manifest.json)
Tokens = HOW things look | Manifest = WHAT structure exists
